Use these checks in order:

```bash
tvl-check-structural my-spec.tvl.yml
tvl-check-operational my-spec.tvl.yml --json
```

- `tvl-check-structural` asks whether at least one valid configuration exists.
- `tvl-check-operational` asks whether the current `env.context.*` assumptions let the study proceed.

If those pass, add targeted tests so future edits do not quietly break the contract. For unit testing
in Python, use the TVL SDK:

```python
from tvl import loader, structural_sat

def test_constraints_satisfiable():
    """Verify the spec has at least one valid configuration."""
    module = loader.load("my-spec.tvl.yml")
    result = structural_sat.check(module)
    assert result.satisfiable, f"UNSAT: {result.unsat_core}"

def test_specific_config_valid():
    """Verify a specific configuration passes constraints."""
    module = loader.load("my-spec.tvl.yml")
    config = {"model": "gpt-4o-mini", "temperature": 0.3, "max_tokens": 512}
    errors = structural_sat.validate_config(module, config)
    assert not errors, f"Constraint violations: {errors}"
```

Run the tests:

```bash
pytest test_constraints.py -v
```

<ConceptCallout title="Cheap Checks First" tone="tip">

The whole point of these tools is to catch broken contracts before a tuner, evaluator, or CI job burns
time on them. Run the static and operational checks long before you discuss promotion.

</ConceptCallout>
